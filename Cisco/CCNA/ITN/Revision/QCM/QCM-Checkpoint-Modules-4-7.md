# ✅ RÉPONSES ET EXPLICATIONS DÉTAILLÉES

## Question 1
**Réponses correctes : b, c, e**

**Explication :**
La **couche physique (Couche 1)** du modèle OSI est responsable de :
- **b) Encodage des signaux** : Convertir les données en signaux transmissibles (électriques, optiques, radio)
- **c) Transmission de bits** : Transport physique des bits sur le média
- **e) Génération de signaux** : Création des signaux électriques (cuivre), optiques (fibre) ou radio (sans fil)

Les autres fonctions appartiennent à des couches supérieures :
- a, f) Couche 3 (Réseau)
- d) Couche 2 (Liaison de données)

---

## Question 2
**Réponse correcte : b**

**Explication :**
Un **câble croisé (crossover)** est utilisé pour connecter directement deux appareils similaires :
- Ordinateur à ordinateur
- Switch à switch
- Routeur à routeur

Le câble croisé inverse les paires de transmission (TX) et réception (RX) permettant la communication directe.

**Câble droit** : Connecte des appareils différents (ordinateur à switch, switch à routeur).

**Note** : Avec Auto-MDIX (détection automatique), les appareils modernes peuvent fonctionner avec les deux types.

## Question 3
**Réponse correcte : c**

**Explication :**
Normes Ethernet sur cuivre :
- **10BASE-T** : 10 Mbps, UTP Cat 3, 100 m
- **100BASE-TX** : 100 Mbps, UTP Cat 5, 100 m (Fast Ethernet)
- **1000BASE-T** : 1 Gbps (1000 Mbps), UTP Cat 5e/6, 100 m (Gigabit Ethernet)
- **10GBASE-T** : 10 Gbps, UTP Cat 6a/7, 100 m

La norme **1000BASE-T** (Gigabit Ethernet) fournit 1 Gbps sur câble en cuivre jusqu'à 100 mètres.

## Question 4
**Réponse correcte : b**

**Explication :**
Structure d'un câble à fibre optique :
- **Cœur (core)** : Partie centrale en verre ou plastique qui transporte réellement les signaux lumineux
- **Revêtement (cladding)** : Couche entourant le cœur, réfléchit la lumière pour la maintenir dans le cœur
- **Gaine protectrice (jacket)** : Protection externe

Le signal lumineux se propage dans le **cœur** par réflexion totale interne.

## Question 5
**Réponse correcte : b**

**Explication :**
Différences entre fibre monomode (SMF) et multimode (MMF) :

**Monomode (SMF - Single-Mode Fiber)** :
- Cœur très petit (8-10 µm)
- Distances *très longues* (jusqu'à 100+ km)
- Utilise des *lasers*
- Plus coûteuse

**Multimode (MMF - Multi-Mode Fiber)** :
- Cœur plus large (50-62.5 µm)
- Distances moyennes (quelques centaines de mètres)
- Utilise des *LED*
- Moins coûteuse

La principale différence est que **SMF permet des distances beaucoup plus longues** (b est correct).

## Question 6
**Réponse correcte : a**

**Explication :**
Les **ondes radio** (communications sans fil) sont vulnérables à :
- **Obstacles physiques** : Murs, mobilier, bâtiments réduisent le signal
- **Interférences électromagnétiques (EMI)** : Autres appareils sans fil, micro-ondes, Bluetooth
- **Interférences radio (RFI)** : Autres équipements radio
- **Atténuation** : Perte de signal avec la distance

C'est pourquoi la portée et la fiabilité du Wi-Fi peuvent varier considérablement selon l'environnement.

## Question 7
**Réponse correcte : b**

**Explication :**
Dans un environnement avec de fortes **interférences électriques** :
- **STP (Shielded Twisted Pair)** : Paires torsadées avec blindage métallique protégeant des EMI/RFI
- **UTP (Unshielded Twisted Pair)** : Pas de blindage, vulnérable aux interférences

Le câble **STP** offre une meilleure protection contre les interférences électromagnétiques grâce à son blindage.

Alternative : La **fibre optique (d)** est totalement immune aux EMI mais plus coûteuse.

## Question 20
**Réponse correcte : b**

**Explication :**
Une **trame Ethernet** est l'unité de données de la **couche 2 (Liaison de données)** comprenant :
- **En-tête (Header)** : Adresse MAC destination, adresse MAC source, Type/Longueur
- **Données (Payload)** : Paquet IP encapsulé (46-1500 octets)
- **Trailer** : FCS (Frame Check Sequence) pour détection d'erreurs

Structure trame Ethernet II :
```
[ Préambule | SFD | MAC Dest | MAC Source | Type | Données | FCS ]
```

Paquet IP (c) et segment TCP (c) sont des couches supérieures.

## Question 21
**Réponse correcte : b**

**Explication :**
Taille d'une trame Ethernet II (sans VLAN 802.1Q) :
- **Minimum** : **64 octets** (de l'adresse MAC destination au FCS)
- **Maximum** : 1518 octets

Si les données sont < 46 octets, du padding (rembourrage) est ajouté pour atteindre 64 octets minimum.

**Détail** :
- En-tête : 14 octets (6 + 6 + 2)
- Données : 46-1500 octets
- FCS : 4 octets
- Total : 64-1518 octets

**Note** : Le préambule (8 octets) et SFD ne sont pas comptés dans la taille de trame.

## Question 23
**Réponse correcte : b**

**Explication :**
Le rôle principal d'un **switch Ethernet** est de **transférer les trames basées sur les adresses MAC** (couche 2).

Fonctionnement :
1. Reçoit une trame sur un port
2. Examine l'adresse MAC de destination
3. Consulte sa table d'adresses MAC
4. Transfère la trame uniquement sur le port approprié

Différence avec routeur :
- **Switch** : Couche 2, adresses MAC, même réseau
- **Routeur** : Couche 3, adresses IP, entre réseaux différents

## Question 24
**Réponse correcte : a**

**Explication :**
Un switch apprend les adresses MAC dynamiquement en **examinant l'adresse MAC source** des trames reçues :

Processus d'apprentissage :
1. Trame arrive sur port 1
2. Switch lit l'**adresse MAC source**
3. Ajoute l'association "MAC source → Port 1" dans sa table MAC
4. Utilise cette information pour futures transmissions

Table MAC (CAM - Content Addressable Memory) :
```
MAC Address       Port    Age
00:1A:2B:3C:4D:5E  Fa0/1   5 sec
00:AA:BB:CC:DD:EE  Fa0/2   10 sec
```

## Question 25
**Réponse correcte : b**

**Explication :**
Un **domaine de collision** est une zone réseau où deux appareils peuvent transmettre simultanément et provoquer une **collision** (corruption des données).

**Avec hubs** :
- Tous les ports partagent le même domaine de collision
- Utilisation de CSMA/CD (Carrier Sense Multiple Access with Collision Detection)

**Avec switches** :
- Chaque port = domaine de collision séparé (en mode full-duplex)
- Pas de collisions en mode full-duplex

**Note** : Un domaine de broadcast (a) est différent - zone où les broadcasts sont propagés.

---

## Question 26
**Réponse correcte : b**

**Explication :**
Les **switches Ethernet modernes éliminent les collisions** car :
- Chaque port fonctionne en **mode full-duplex** (transmission et réception simultanées)
- Chaque port constitue un **domaine de collision séparé**
- Micro-segmentation : Communication point-à-point entre switch et appareil

**Comparaison** :
- **Hub** (ancien) : Half-duplex, collisions possibles, un seul domaine de collision
- **Switch** (moderne) : Full-duplex, pas de collisions

CSMA/CD n'est plus nécessaire avec les switches full-duplex.

---

## Question 27
**Réponse correcte : a**

**Explication :**
Un **domaine de broadcast** est une zone réseau où les **trames broadcast** (adresse destination FF:FF:FF:FF:FF:FF) sont propagées à tous les appareils.

Caractéristiques :
- Les switches propagent les broadcasts sur tous les ports
- Les **routeurs segmentent les domaines de broadcast** (ne propagent pas les broadcasts)
- Un VLAN = un domaine de broadcast

Exemple : Requête ARP (broadcast) est reçue par tous les appareils du domaine de broadcast.

---

## Question 28
**Réponse correcte : b**

**Explication :**
Lorsqu'un switch ne connaît pas l'adresse MAC de destination (pas dans sa table MAC), il utilise le **flooding** (inondation) :

Processus :
1. Switch reçoit une trame avec MAC destination inconnue
2. Envoie la trame en **broadcast sur tous les ports** sauf celui d'origine
3. L'appareil avec cette MAC répond
4. Switch apprend l'association MAC-Port pour futures transmissions

C'est aussi le comportement pour les **trames broadcast** (FF:FF:FF:FF:FF:FF).

## Question 30
**Réponse correcte : b**

**Explication :**
La méthode **store-and-forward** (stocker et transmettre) :

Processus :
1. Switch **reçoit la trame entière** dans sa mémoire tampon
2. **Vérifie le FCS** (Frame Check Sequence) pour détecter les erreurs
3. Si la trame est valide, la **transmet** au port de destination
4. Si erreur détectée, **rejette** la trame

**Avantages** : Fiabilité, détection d'erreurs
**Inconvénient** : Latence légèrement plus élevée

**Autres méthodes** :
- **Cut-through** (a) : Transmet dès lecture de l'adresse MAC destination (plus rapide, pas de vérification d'erreurs)
- **Fragment-free** : Compromis entre les deux

Store-and-forward est la méthode la plus fiable et la plus couramment utilisée.
