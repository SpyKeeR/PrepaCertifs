## 📖 Programme de l'examen

L'examen Linux Essentials couvre **5 sujets principaux** :

### Topic 1 : The Linux Community and a Career in Open Source (8%)
**Poids : 8 points sur 100**

- **1.1** Évolution et distributions Linux populaires
  - Distributions : Debian, Ubuntu, Red Hat, Fedora, SUSE, openSUSE
  - Systèmes embarqués et Android
  - Linux dans le cloud
  
- **1.2** Applications open source majeures
  - Applications bureautiques (LibreOffice, OpenOffice)
  - Navigateurs (Firefox, Chromium)
  - Serveurs (Apache, Nginx, MySQL, PostgreSQL)
  - Partage de fichiers (Samba, NFS)
  - Langages de programmation
  
- **1.3** Logiciels open source et licences
  - Licences : GPL, LGPL, BSD, Creative Commons
  - Free Software Foundation (FSF)
  - Open Source Initiative (OSI)
  
- **1.4** Compétences ICT et travail sous Linux
  - Environnements de bureau (GNOME, KDE)
  - Ligne de commande vs interfaces graphiques
  - Sécurité et bonnes pratiques

### Topic 2 : Finding Your Way on a Linux System (10%)
**Poids : 10 points sur 100**

- **2.1** Bases de la ligne de commande
  - Shells (Bash, sh, csh)
  - Commandes de base : `ls`, `cd`, `pwd`, `echo`
  - Variables d'environnement
  - Chemins absolus et relatifs
  
- **2.2** Utiliser la ligne de commande pour obtenir de l'aide
  - Pages man (`man`)
  - Documentation info (`info`)
  - Fichiers `/usr/share/doc`
  - Option `--help`
  
- **2.3** Navigation dans les répertoires et liste des fichiers
  - Structure du système de fichiers
  - FHS (Filesystem Hierarchy Standard)
  - Répertoires importants : `/etc`, `/home`, `/var`, `/tmp`
  - Options de `ls` : `-l`, `-a`, `-h`, `-R`
  
- **2.4** Création, déplacement et suppression de fichiers
  - `touch`, `mkdir`, `cp`, `mv`, `rm`, `rmdir`
  - Globbing et wildcards (`*`, `?`, `[]`)
  - Fichiers et répertoires cachés

### Topic 3 : The Power of the Command Line (10%)
**Poids : 10 points sur 100**

- **3.1** Archivage de fichiers
  - Commande `tar` (création, extraction, compression)
  - Compression : `gzip`, `bzip2`, `xz`, `zip`
  - Archives : `.tar.gz`, `.tar.bz2`, `.tar.xz`
  
- **3.2** Recherche et extraction de données
  - Redirections : `>`, `>>`, `<`, `|`
  - `grep` et expressions régulières
  - `head`, `tail`, `wc`, `sort`, `cut`
  - STDIN, STDOUT, STDERR
  
- **3.3** Transformation de commandes en scripts
  - Shebang (`#!/bin/bash`)
  - Variables et paramètres
  - Structures conditionnelles (`if`, `else`)
  - Boucles (`for`, `while`)
  - Permissions d'exécution (`chmod +x`)
  - Codes de sortie

### Topic 4 : The Linux Operating System (8%)
**Poids : 8 points sur 100**

- **4.1** Choix d'un système d'exploitation
  - Distributions : Enterprise, Consumer, Embedded
  - Différences Windows/Linux/Unix
  - Cas d'usage
  
- **4.2** Comprendre le matériel informatique
  - CPU, RAM, périphériques de stockage
  - Commandes : `lscpu`, `lsblk`, `free`, `/proc/cpuinfo`
  - `/dev` et fichiers de périphériques
  
- **4.3** Où les données sont stockées
  - Système de fichiers : `/`, `/home`, `/etc`, `/var`, `/tmp`, `/usr`
  - Fichiers de configuration
  - Processus et `/proc`
  - Swap et mémoire virtuelle
  
- **4.4** L'ordinateur sur le réseau
  - Modèles OSI et TCP/IP
  - Adressage IPv4 et IPv6
  - DNS, DHCP, NAT
  - Commandes : `ip`, `ping`, `dig`, `ss`
  - Routage et passerelles

### Topic 5 : Security and File Permissions (7%)
**Poids : 7 points sur 100**

- **5.1** Sécurité de base et identification des types d'utilisateurs
  - Utilisateurs et groupes
  - UID et GID
  - Fichiers `/etc/passwd`, `/etc/group`, `/etc/shadow`
  - Commandes : `whoami`, `id`, `who`, `w`
  
- **5.2** Création d'utilisateurs et de groupes
  - `useradd`, `userdel`, `usermod`
  - `groupadd`, `groupdel`
  - `passwd`
  
- **5.3** Gestion des permissions et propriété de fichiers
  - Permissions : `r`, `w`, `x`
  - Propriétaire, groupe, autres
  - `chmod` (numérique et symbolique)
  - `chown`, `chgrp`
  - Bits spéciaux : SUID, SGID, Sticky bit
  
- **5.4** Répertoires et fichiers spéciaux
  - Liens symboliques et liens durs (`ln`)
  - Fichiers spéciaux : `/dev/null`, `/dev/zero`, `/dev/random`
  - Sockets et pipes

**Total : 43 points répartis**

## 💡 Méthodologie d'étude recommandée

### Phase 1 : Apprentissage initial (2-3 semaines)

1. **Lire le cours complet** (`Cours-Preparation.md`)
   - Prendre son temps pour comprendre chaque concept
   - Noter les points qui semblent difficiles
   - Pratiquer les commandes au fur et à mesure

2. **Pratiquer sur un système Linux réel**
   - Installer une distribution (Ubuntu, Debian, Fedora)
   - VM (VirtualBox, VMware) ou WSL2 sur Windows
   - Exécuter toutes les commandes du cours
   - Créer des scripts simples

3. **Consulter les notes ciblées** (`Notes-Revision-Sujets-Cibles.md`)
   - Approfondir les points complexes
   - Compléter sa compréhension

### Phase 2 : Révision et consolidation (1-2 semaines)

4. **Réviser avec la fiche synthétique** (`Fiche-Revision-Synthetique.md`)
   - Lecture rapide quotidienne
   - Vérifier qu'on maîtrise chaque point
   - Identifier les lacunes

5. **Tester ses connaissances** (`QCM-Linux-Essentials.md`)
   - Faire le QCM sans regarder les réponses
   - Vérifier les réponses
   - Comprendre les erreurs
   - Reprendre le cours sur les points faibles
   - **Objectif : 90%+ de bonnes réponses**

6. **Utiliser Anki pour la mémorisation espacée**
   - Importer les 3 decks dans Anki
   - Réviser 20-30 cartes par jour
   - Système de répétition espacée pour mémorisation long terme

### Phase 3 : Préparation finale (1 semaine)

7. **Révision intensive**
   - Relire la fiche synthétique plusieurs fois
   - Refaire le QCM jusqu'à obtenir 95%+
   - Continuer les révisions Anki quotidiennes

8. **Pratique finale**
   - Labs pratiques : créer utilisateurs, gérer permissions
   - Scripts shell : conditions, boucles, variables
   - Navigation système de fichiers sans GUI
   - Archivage et compression de fichiers

9. **Simulation d'examen**
   - Conditions réelles : 40 questions en 60 minutes
   - Sans aide extérieure
   - Analyser les erreurs

### Le jour de l'examen

- Bien dormir la veille
- Arriver en avance (ou préparer son espace si examen en ligne)
- Lire chaque question attentivement
- Utiliser le temps restant pour vérifier les réponses
- Ne pas passer trop de temps sur une question difficile

## ✨ Conseils pour réussir

### Avant l'examen

✅ **Pratique régulière**
- Utiliser Linux quotidiennement
- Expérimenter avec les commandes
- Ne pas avoir peur de "casser" (dans une VM !)

✅ **Comprendre, pas mémoriser**
- Comprendre la logique derrière les commandes
- Savoir pourquoi, pas seulement comment
- Les concepts restent, la mémorisation s'oublie

✅ **Utiliser les ressources**
- Pages man : `man commande`
- Option --help : `commande --help`
- Google et Stack Overflow pour exemples

### Pendant l'examen

✅ **Gestion du temps**
- 60 minutes pour 40 questions = 1,5 minute par question
- Ne pas rester bloqué sur une question
- Marquer pour révision et revenir plus tard

✅ **Lire attentivement**
- Bien lire chaque question
- Attention aux mots comme "NOT", "EXCEPT"
- Toutes les réponses possibles peuvent sembler correctes

✅ **Élimination**
- Si incertain, éliminer les réponses évidemment fausses
- Affiner entre les réponses restantes
- Utiliser la logique Unix/Linux

### Après l'examen

✅ **Si réussite**
- Célébrer ! 🎉
- Mettre à jour CV et LinkedIn
- Partager sur réseaux sociaux (#LPICertified)
- Commencer à préparer LPIC-1

✅ **Si échec**
- Ne pas se décourager
- Analyser les domaines faibles
- Reprendre la préparation ciblée
- Repasser l'examen (pas de limite de tentatives)

## 🆘 FAQ

### Combien coûte l'examen ?
**~120-150 USD** selon la région (tarif LPI standard, Code promo dispo à la fin du cours "Linux Essentials" sur NetAcad)

### Peut-on passer l'examen en ligne ?
**Oui**, via OnVUE (surveillance en ligne) ou dans un centre Pearson VUE

### Faut-il connaître plusieurs distributions ?
**Non**, la certification est vendor-neutral. Les concepts sont universels.

### Quel système Linux utiliser pour pratiquer ?
**Ubuntu 22.04 LTS** ou **Debian 12** sont excellents pour débuter

### Les commandes sont-elles les mêmes partout ?
La plupart oui, mais certaines options peuvent varier. L'examen teste les concepts standards.

### Faut-il être développeur ?
**Non**, cette certification est pour débutants. Aucun prérequis de programmation.

### L'examen est-il difficile ?
**Modéré**. Avec une bonne préparation (60-80h), le taux de réussite est élevé.

### Puis-je utiliser des ressources pendant l'examen ?
**Non**, c'est un examen fermé (closed-book). Aucune documentation autorisée.

## 📌 Points clés à retenir

### Commandes essentielles à maîtriser

```bash
# Navigation
cd, pwd, ls, tree

# Fichiers
touch, mkdir, cp, mv, rm, rmdir, cat, less, more

# Recherche
find, locate, grep, which, whereis

# Permissions
chmod, chown, chgrp, umask

# Utilisateurs
useradd, userdel, groupadd, passwd, id, whoami, who, w

# Archives
tar, gzip, bzip2, xz, zip, unzip

# Réseau
ip, ping, dig, ss, netstat, route

# Système
df, du, free, top, ps, kill, uname, hostname

# Aide
man, info, --help, apropos
```

### Fichiers et répertoires critiques

```
/etc/passwd          # Comptes utilisateurs
/etc/group           # Groupes
/etc/shadow          # Mots de passe hachés
/etc/hosts           # Résolution DNS locale
/etc/fstab           # Montage systèmes de fichiers
/proc/               # Informations processus
/dev/                # Fichiers périphériques
/tmp/                # Temporaire (effacé)
/var/log/            # Journaux système
/home/               # Répertoires utilisateurs
```

### Permissions en octal

```
7 = rwx (4+2+1)
6 = rw- (4+2)
5 = r-x (4+1)
4 = r-- (4)
3 = -wx (2+1)
2 = -w- (2)
1 = --x (1)
0 = --- (0)

Exemples :
755 = rwxr-xr-x
644 = rw-r--r--
777 = rwxrwxrwx
600 = rw-------
```
